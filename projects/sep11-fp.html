SEP 11 Freedom Project

Context

I am a student at HSTAT in the Software Engineering Program. The "Freedom Project" for SEP11 is a year-long project all about making something using JavaScript along with a third-party JS tool.

Content

For my project, I chose to independently study gDevelop in order to help me make a game.

Reflection

Challenges

A challenge that I had was getting the aiming system to work.
This is the case because I wanted to have a trajectory line.
However, putting the same exact values for both the player and the aim object does not result in them overlapping.
Instead, the only way to get it to work is to throw random values in until they line up to a decent extent.

Another challenge I had was implementing a background into my game.
This was difficult since in gDevelop, to make a background, you must use a tiled sprite, not a normal sprite.
This is significant since images from google are very small, so to get a background to cover the entire screen without repeating, the image itself must be resized to match the canvas.
The tool I used has trouble doing this and I ended up with a pixelated background.

Takeaways

A takeaway that I have from doing my Freedom Project is that simplicity is oftentimes better than complexity.
I realized this when I swapped my sequence of three different conditionals to only using one
I decided to do this because although using three conditionals does make the trajectory line more accurate, it isn't very smooth when one intersection point is reached (changing from using one conditional to using another) since it would skip up due to a different force being applied.

Another takeaway I have is that you should check the documentation of the tool before committing.
I didn't do this so I ended up picking gDevelop, a tool that barely uses any javascript.
Even worse is that SEP 11 is all about javascript, so to have a year long project that barely has any javascript in it is a bit....

Next Steps

If I had more time, I would try to adjust the hitbox of the flag and fix the hitbox of the ball (it broke for some reason).
These aren't major errors, but the broken flag hitbox makes it so that it's sometimes very hard to change scenes.
